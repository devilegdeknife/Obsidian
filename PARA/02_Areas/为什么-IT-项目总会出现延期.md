大卫·卡特勒（David Cutler）是比尔盖茨亲自参与挖墙角挖过来的牛人。

卡特勒觉得dos是个玩具，unix系统像个操作系统但是设计很一般，相当不完善。卡特勒加入微软之后，还带了20多个旧部下，他打算开发一个支持多种CPU指令，能运行在PC上的真正多任务操作系统，他希望这个操作系统能让pc机像服务器一样稳定，他写完spec之后，给比尔盖茨预估的时间：1.5年。后来，这个操作系统有个名字叫NT。

卡特勒手下的人都是软件高手，对服务器非常精通但是对pc架构知道得不多，于是微软给他安排了熟悉pc的软件高手，这些高手在卡特勒眼中，水平都相当差。微软内部很多员工都觉得卡特勒那帮人好像有点瞧不起人，后来配合起来出各种问题。

卡特勒遇到的第一个问题就是intel公司新版的CPU还没真正生产出来，他们手头只有intel公司给的新版本CPU指令集和各种参数指标。这当然拦不住卡特勒的团队，他们很快写了个虚拟程序来模拟新版本cpu，然后基于这个模拟的cpu来写程序，但是模拟的cpu速度相当慢，程序出问题的时候，也不知道是自己的代码出问题还是虚拟cpu的代码出问题。

几个月之后，卡特勒拿到intel公司给的真正cpu，团队差点吐血，因为指令集更新了，他们前面写的好大一部分代码全部作废。

一年半过去了，NT还没法独立运行，硬件又升级了，新的cpu指令也要求支持，因为微软和intel公司有深度合作。

鲍尔默负责微软全球市场的，他把市场部的需求列成清单给卡特勒，卡特勒明确表示项目要延期，当然，一切都在可控之中，最多八个月之后就能正常发布。比尔盖茨看了新的操作系统，非常激动，认为这将彻底改变服务器操作系统。

windows一开始不是操作系统而是作为应用程序运行在dos操作系统之上，很不成功。微软更新了多个版本，win3.1忽然大获成功。

卡特勒开始设计NT的时候要求兼容dos，后来要求兼容IBM的OS2，后来要求支持图形界面等等。需求每天都在增加。NT一开始设计的最低运行内存是8M，要把这么多东西加进去，卡特勒要求每个程序员都得严格预估自己那部分需要多少内存。

nt慢慢开始成型，终于有一天，nt能独立运行。

负责开发打印机的程序员发现死活打印不出东西，卡特勒鄙视了这个程序员一顿之后，宣布3天内搞定。卡特勒亲自写了代码，花了一周时间都没搞定。长期被卡特勒压制的程序员终于逮到机会，天天嘲笑卡特勒，卡特勒实在受不了，找几个高手来帮忙查看，几位软件高手检查了一周，没发现任何问题，最后有人无意中发现链接打印机的那根数据线电压不够，换了根线就好了，卡特勒的代码一开始就是对的。

win3.11的后续测试版本发行，大获成功，比尔盖茨宣布NT改名叫windows NT，api全面兼容windows，于是需求再次发生变化。

c++忽然开始流行起来，当时开发NT主要用汇编和c，部分程序员听说c++的牛逼特性之后，决定用c++开发，卡特勒对c++有点担心但并没反对，负责每日构建（daily builder）的人每天从服务器上获取代码开始编译构建。c++代码很快就让他们崩溃，速度慢问题多开发工具不成熟等等，经常让构建失败。卡特勒经常去和构建人员一起，每天心惊肉跳担心builder失败，一旦失败卡特勒就开始疯狂的F**K这个程序员。有人建议在房间装上钢门以免被卡特勒爆。

卡特勒不信任微软工具开发组的人，他多次查看c++编译的汇编代码。在NT发行前夜，刻光盘之前，还在做这个事情。

windows nt开始开发是1988年，发布的第一个版本是3.1 ，1993年发布，总共用了5年多时间，项目延期了好几倍。比较搞笑的是，windows 也大幅度延期，在1995年才发布，名字也改名叫win95。

windows nt发布的时候没引起什么轰动，因为不是针对个人用户的，win95发行之后，引起抢购狂潮，很多粉丝排队购买。

win95的后续版本到win2k好像就结束了，后来大家用的windows，内核是卡特勒负责开发的NT，一直延续到现在。

比较搞笑的是，windows nt没有打败当时预设的竞争对手unix，而是把netware给弄没了，unix后来被其变种linux给弄得几乎没市场。

总结一下这个项目延期的原因

1，需求发生变化；

2，需求增加

3，人员配合度不够

4，钱太多。

5，开发工具不成熟

说一下4.

正常公司，项目延期会出现亏损，然后被砍，钱足够多，会导致项目不断延期但不会轻易被砍掉。微软把这个项目当成微软的未来，所以项目肯定不会被砍，nt第一版花了1.5亿，后来这个系统给他们赚了上千亿。

随便写的，有点乱，就这样吧。
