# 为什么 Vue 3 设计了那么多重复功能的 API？
[内容链接](https://www.zhihu.com/question/1933969813643989014/answer/1935432735381525060)

简单来说就是历史兼容，Vue 2 是 2016 年发布的，到今天九年，只有一次 breaking change 的机会。生产环境里成千上万的 app 在跑，所以 3.x 即使一个新 API 是为了彻底代替旧 API 而存在，旧 API 也不能删，因为一定有谁的 app 依赖了这些旧 API。

Semantic versioning 意味着除非出 Vue 4，不然不能删任何旧 API。要是出 Vue 4，肯定会出来一批骂娘的，说又更新，学不动了，旧 API 好好的非要给他们增加工作量；要是 minor 里面删，你看看那些 CI 崩了的人会不会骂得更难听。

Vue 3 现在的最佳实践就是，用不到的你就当它不存在就好。经常看见这种 API 太多的论调我很怀疑是不是国内一些培训班或者是卖课的为了撑内容强行把一些其实用不到的东西也当八股一样硬塞给初学者了。

当然在各位知乎大神眼里这都是设计失误，毕竟各位都是吸收了过去十年整个前端所有知识精华的高人，我五年前的思想水平没有达到各位今天的水准真是太惭愧了。

你看，只要在一个领域做得足够久，不管你做什么都会被后来人嘲讽的。
