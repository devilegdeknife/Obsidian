# 为什么现在吹Rust的人这么多？
[内容链接](https://www.zhihu.com/question/624623198/answer/3685475173)

开门见山：**Rust 是积累了人类在软件工程学，编译原理等领域上数十载理论和工程细节智慧的结晶。**是一门在学习熟练后，很难找到缺点的编程语言。如果你学习过10种以上的编程语言，并都做过项目，那么你会在 Rust 上面找到很多其他编程语言的影子。

Rust 的整体设计相比于 C++，可以以更少的人月完成底层软件的开发，并且只要你遵守开发规范，开发出的软件不会因为野指针满天飞而内存泄漏；只要编译器不报错，放心地无畏并发；就算是出现 panic，八成也是因为你为了早点下班乱用 unwrap，rust release 模式下的 build 还会直接告诉你出错地点在哪一个文件的哪一行哪一列，生怕你太菜还在狼狈地找你那该死的调试器。真的，当你开始习惯调试器，计算机的很多领域就与你无缘了。

开发者不再需要再折腾那该死的写成绝对路径的 SB Makefile，然后去项目的 issue 上用机翻的英文国粹口吐芬芳；公司也不再需要花费高额开支雇佣”经验老道“的程序员，仅仅因为他们可以完成其他人无法完成的依赖项配置和跨平台编译，现在一个 cargo build --release 直接完事。

![图片描述](https://picx.zhimg.com/50/v2-de658642c6b11f6ad92149ed549d0577_720w.jpg?source=2c26e567)

而且经过数十年的发展，Rust 的社区生态已经非常完善了，你连 system verilog 的原型解析器框架都能找到，还不止一个。基本上干底层要的组件，Rust 大部分都有，这也为 ”Rust 重写一切“ 的现象提供了技术基础。

Rust 编译器已经完成了自举，大量的底层组件安装后可以直接修改源代码，对于很多场景的高度定制化开发这是非常重要，不像某些编程语言，一路定义跳转，最终只到一个不明所以的存根文件，Cargo.toml 里面指定一下 package 的 path， 和 git submodule 完美配合。

我一个人工智能研究者兼前端爱好者都知道 safe tensor / wasm 驱动的 egui 框架底层都是 rust 写的。最新的 webgpu 据说也是 rust 写的。你就可以看到 Rust 会如何慢慢影响计算机目前的软件栈生态。

搞嵌入式的朋友要站出来反驳了：为啥我们公司还在用C/C++？因为厂商给的电机/驱动板/芯片的驱动库或者SDK interface大部分都是 C/C++，硬件领域的技术栈变动往往非常慢，因为成本敏感。而且大部分的浅层嵌入式其实就是把数字翻译一下输入电机，做个调制解调，用不到 Rust。不过放心，未来我制作的板子和FPGA，一定会提供 Rust 的 SDK 的。

还是那句话，Rust 适合开发底层软件，嵌入式的驱动程序和顶层的人工智能胶水语言不在我的讨论范围内，要吵架请出门左拐。

> 2025.5.19，经群友 Kenji Mouri 补充，Rust 无法进军嵌入式还有如下几点：
> - 厂家没法手搓能用的 Rust 编译器，但是不想公开细节，更不想适配 LLVM 和 GCC 这种复杂框架
> - Rust 编译出来的二进制体积相对于 C/C++ 对 MCU 事务不友好
> - Rust 的原生实现的底层库没有，或者即使有也质量堪忧（譬如复杂嵌入式设备常用的文件系统解析库等）
> - 还有就是 Rust 的语言特性对于裸机整活很可能得开 nightly，而大部分人不会冒险用 nightly

入门门槛的话，我个人认为 Rust 相对于其他常用编程语言对新手非常不友好，入门速度应该是常规编程语言中最慢的那一个，如果你是整天无所事事的带学生，真心建议你有空学习 Rust。我当初学习 java 到上手开始写 CURD，大概花了半天时间，但是 Rust，从我部署中文版本的 Rust 教程，到我开始入手写一个含有大量线程同步的形式语言解析服务器和异步调度器，花了整整三天，我一开始以为我可以像写 Java，C#，Swift 这些语言一样，只花半天学习就能入门的，结果 cargo checker 的各种我根本无法用大模型解决的报错直接教我重新做人——Rust，它很不一样。

![图片描述](https://picx.zhimg.com/v2-ad78e8f8702177fa9ecc85a00412193d_r.jpg?source=2c26e567)

然后嘛。。。然后我就去乖乖学习了，突然想起大一C++老师的话，再优秀的程序员也需要秉持一颗学徒的心，这下我是真的明白了。

最后说点题外话，我认为 Rust 和 Typescript 这些语言不太可能会在短时间内于国内普及，原因很简单，有些人需要养家糊口，于是会主动地去网上造势为自己的身份和位置挽尊；一家技术公司进行大规模技术栈变更的代价很多小朋友可能也不太清楚。

这也是可以理解的，因此，好东西阁下自己和自己的小团队用用就行，没必要去顶撞你们的前辈。别用我的文章或者别人的文章作为反驳的依据，用自己的作品和社区影响力作为反驳他人的依据。
