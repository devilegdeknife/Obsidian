# AV1和HEVC的性能差异有多大？
[内容链接](https://www.zhihu.com/question/344725190/answer/1902679122746020689)

低质量提升很大，高质量提升就很一般了。

实测很多720P甚至1080P的视频，可以用AV1 CRF 48直接压，质量无明显损失，当然前提是本身就是低质量视频，720P不代表低质量，关键这个720P或1080P是通过强拉的那种，本身看上去都是模糊的锯齿的，那48足以，甚至可以用到52也不会有太大损失，H265压片基本上超过26就没法看了，实际压265的时候基本上是CRF 26~19，动漫23，真人21，收藏级真人电影19

现在我批量把网盘存的视频用AV1转压，基本上那些1.3G的电视剧能压到500M左右，性价比十分高。

至于不少人说H264/265差距不大，那是忽略了网络上乱七八糟的压法，我习惯下载电视剧，尤其是老电视剧，实际网络上的电视剧十有八九都是H264且直接放大到1080P/4K用固定码率压的，压出来那个大，锯齿那个满，这种片一眼480P的源，基本上能压到原体积的1/4体积，没有明显质量损失；而质量很好的3G/集的片，CRF26压到800M就差不多到头了，能压那么多也得益于之前就是码率冗余。

以下是一些压片经验。

1、一部电视剧，如果全部都是一个大小的，比如45分钟电视剧，每一集都是1.3G的，可压。顺带一提，这种恒定码率的大概率就是用NVENC硬压的，速度很快，质量嘛，只能说码率高的还可以，所以大部分人压都直接上高码率，CRF进行转压哪怕同样264都能省出不少空间来。

2、原始格式为H264/AVC的，可压，且性价比高；原始格式为H265/HEVC的，且是4K的，参考第一条。

3、肉眼看质量，如果都是拉胯质量的，直接48~42往死里压，基本上能压到1/3~1/4。

4、收藏要求且原版质量十分高，看原版是否H264/AVC，如果是同样结合第一条，都满足后，CRF36~26压，当然，网盘足够大的也可以不压；拿我上面3G压到800多的为例，就是同一帧肉眼对比人脸纹理质量不变。如果你拉到最高质量还是比不上X265不要觉得奇怪，SVT-AV1并不是全面超越265的，从各大网站评测可见，**SVT-AV1的优势区间是5000码率以下，码率超过5000的265且是CRF压的，你转压AV1大概率会损失画质，收藏级的资源大可不必。其中压动画提升最大，因为动漫人物普遍脸部纹理不足，只有有限的线条勾勒出纹路，其中旧视频RV40的纹路还有锯齿或者说噪点，AV1反而因为模糊自带抗锯齿，类似AI修复，而且转压MP4封装的AV1视频不会像老RMVB那样跳跃进度条后有1秒左右的“失声”。**

5、H265常用CRF为26~19（低于19体积大于264），AV1常用CRF为（48~26），低于26个人认为价值不大，大多数情况下都会比原文件还大。

6、不用一次性压全部，可以挑一集出来先试着参数压一次，判断质量和大小是否需要进一步改良。

7、简单分析，基本上40分钟一集恒定大于800M，非收藏级恒定大于600M就是有压缩空间的，网上95%都是AI修复甚至直接无脑拉分辨率的，糊成狗还弄得一部40集电视剧超过50G，完全就是浪费空间，实际对比过腾讯视频的《法证先锋6》，那是550M就已经4K了，人物脸部纹理清晰可见，用的还只是X265，理论上AV1还能在这个基础上压个10%~20%，当然因为本身比较小且清晰度足够（源是好的），重压AV1显得有点多余，这种我不会重压。

8、风景类纪录片不建议压，比如《地球脉动》、《蓝色星球》，十分难把握，且AV1的高码率其实并不出彩，分分钟压到比源还大，那就没必要了。

9、有人希望有一个能适用所有视频的参数，实际是不存在的，具体每一部剧原本的压缩方式不一样，那你要对其进行二次压缩的参数自然也不可能一样，这也是我前面建议一部剧试着压一集的原因；还是拿法证先锋系列为例，这剧出了6部，其中前三部属于古早的，现在流行在网络上的基本上是强拉分辨率出来的1080P，所以直接48~36（看你自己的接受范围）压下去，能把1.5G压到450M，质量并不会下降太多；但后面3部，普遍压缩得就很好，用26压下去反而变成900M比原来的600M大，用36压到518M也没比原来的600M小多少，且这部剧并不是每一集都600M，最大的有822M，说明它本身就是用CRF压的，也就是说基本上没有冗余码率，虽然AV1确实还能压小，但对于这类压得比较完美且不太大的视频，我个人是不动的。

10、有人说压片电费多，实际上并不会，首先我要测试CPU稳定性必然要各种测试跑全，没有比压片更严苛的测试了，其次我是把CPU降压限制188W功耗的，最后，我强调压的是垃圾画质的片，也就不会用什么P0级压了，都是P10/P8，一部电视剧也就1~4小时以内，完全可以接受。**实际和去年4月同比电费多了15度**（其实算也能算到，压片比挂机高200W功耗，平均一部片2小时就是400W，我压了32部左右，就是400X32/1000=12.8度），这个作为压片代价我觉得完全可以接受的，这也是我强调不压高质量片的原因，你真按专业参数压超高质量片，1080P的一部电影家用CPU能压一个月。

简单总结一下，垃圾透顶的低画质的片，像下面衣领出锯齿的那种，CRF 48+P10（反正是垃圾也就无所谓了），一般的老片，本身模糊但没锯齿的，下面2.43G那种，CRF36+P8；2020年之后质量超高的BDRIP的片（属于收藏级了），CRF26+P6（压好传网盘，本地留一份没压的），合理大小的高质量片，不压，风景纪录片，不压。低画质的动画片一般CRF48+P8，高质量的如《摇曳露营》这类的，不压；AV1高CRF会降低一部分纹理，表现就像4K显示器切换到1080P显示器运行同一个游戏一样，这种差异在没有对比的情况下根本看不出来，而动画片因为本身人脸就缺少纹理，成了最适合AV1高压的视频；简单说除了动画片因为其本身特性能稍微兼顾一下，大部分视频不要想着既要又要，你要压缩就必然损失纹理，AV1能不能压高清保留纹理？按理说能，但肯定不是我上面这些参数，你要自己摸索出来。

一开始其实并没有想着压片，只是想用它测试14900KF的稳定性，结果一发不可收拾。建议至少12核以上的U压，一直不被看好的大小核干这个活很在行。

有人说大硬盘就行，实际我拥有过百T的硬盘，单一台电脑就超过60T，但网盘空间是有限的，上传同样需要时间，大文件既不利于上传也不利于下载，尤其是一些剧情类影片，本身就不是看纹理的，不压缩74G，压缩后10G，回忆一下百度网盘非VIP的下载速度，是不是就有点感觉了？

还有人说自己是做压制的，不用nvenc，但我下的片有不少是人物衣领出锯齿的，我就不知道这种质量是怎么值1.5G，2.43G的，个人认为，现在要压一些长期保存的，非高清的（包括但不限于分辨率是4K但质量不达标那些），都应该尽量AV1压，一个是AV1速度上对比H265并没有太大劣势，shanaencoder使用SVT-AV1支持多线程多任务后，速度完全不虚265，实测同文件AV1-CRF26-P8达到14.73X，X265-CRF23-veryfast速度只有12X（最大达到13.2X）

最后说说这个编码肯定会逐步普及，手机端骁龙8G2已经支持了（2022），PC上NVIDIA的30系也支持了（2020）,40系连编码都支持了（2022），可以说解码这边早就万事俱备，剩下的就是编码那边愿不愿意用了。

我又实际对比了下CRF48压下来的《雪花神剑》双语75G版（1997年的电视剧），确实无论怎么看，画质都丝毫没有损失，1.89G一集压到262M，画质无损失，你能信？但《新三国》双语版（2010年的电视剧）哪怕CRF19都有损失，《一公升眼泪》（2006年电视剧）同理；所以现阶段的SVT-AV1其实是有适用范围的，新电视剧如果没摸索到细节保留的参数，不建议强行压，损失太大；但老电视剧，真的随便压，体积超小还无损失。

某些杠精也是搞笑，明明自己都搞不清概念，就跳出来把水搅浑，硬要杠低质量视频是分辨率虚高否定码率虚高，它肯定没看过腾讯视频的4K，同样的4K腾讯视频的质量比它高出N个档次，但最大不超过500M，这些1.5G/2.4G不是码率虚高是什么？它之所以码率虚高，就是因为源视频低分辨率，它拉分辨率的时候把那些本应模糊处理的锯齿当真实纹理给了更多的码率处理（类似模仿老胶片的噪点信息），导致出来的效果不堪入目，**恰恰就是因为真4K才用到那么高码率，但它保留的数据（锯齿）并不是我们用户需要的**；非专业参数的AV1就把这些用户不需要的数据模糊处理，反而得到了质量更高的视频。（就像游戏中开启SSAA/DLAA一样）

**而真正的分辨率虚高对应的是码率不足**，刚好和我截图的视频完全相反！它不仅码率充足，还把很多“无意义”的信息记录进去。连这些基础的概念都理不清还跳出来抬杠，你就当一辈子杠精吧！这杠精还一个劲的求认同，令人作呕，知道什么叫“答案是丰富多彩的”吗？整天想着否定别人抬高自己，**还是那句话，你要是想表达内容可以自己创作故事，架空世界。**谁说过这话来着？
