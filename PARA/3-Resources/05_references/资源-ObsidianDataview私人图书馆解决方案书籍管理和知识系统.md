# Obsidian+Dataview：私人图书馆解决方案——书籍管理和知识系统
[内容链接](https://zhuanlan.zhihu.com/p/614286218)

## 0. 引言

对于个人藏书较多或阅读时喜欢记录笔记的人来说，图书管理和笔记管理可以说是一个刚需。

市面上虽然有一些图书管理APP，但是功能比较单一，比如我用过一款“私家书藏”，已经三年多没有维护了；最重要的是这类软件有随时停止提供服务的风险，比如之前有很多人用的“晒书房”，一旦不让用了就会存在数据丢失的风险。

还有一些人使用Excel管理自己的图书，把所有数据保存在本地，这样做比较安全。但是如果书籍非常多，录入信息是个很麻烦的事情，而且使用Excel对书籍做分类，功能有一定的局限性。再者，Excel只能管理图书信息，很难满足记录笔记的功能。比如我个人就希望在书籍信息中关联到自己的阅读信息和记录阅读笔记，以便构建自己的知识系统，这就很难用Excel来实现。

由于自己平时使用obsidian，于是基于Obsidian软件+Dataview等插件做了一个个人书籍管理方案。能够实现**个人藏书的信息管理**和**个人知识（笔记）系统的构建**。

这篇文章分享一下搭建方法和步骤，希望给有同样需求的知友提供一些帮助。如果熟悉obsidian和相关插件，那么只需花一两个小时即可完成个人图书馆的搭建。如果不熟悉Dataview，或者还没有使用过obsidian，那么跟着本文的步骤操作，也不难完成。如果嫌麻烦不想自己操作也不要紧，**我会把自己采用的方案打包，包括所有的插件和工具，链接在文章最后，下载下来可以直接使用**（当然需要先安装Obsidian软件）。

但我还是强烈建议跟着这篇文章自己亲手完成搭建过程，过程并不麻烦。通过自己亲手搭建，可以了解一些Dataview的基本使用方法，然后自己探索更多的功能。而不是仅仅局限于我提供的几个功能。

这个方案**适合那些个人藏书较多，或者平时买书较多，希望能掌握自己的藏书、购书情况，对书籍进行各种分类统计和管理；并且希望能够与自己的知识系统关联起来的人。**

## 2. 书籍管理功能实现

在介绍具体步骤之前，先简单介绍以下dataview的基本工作原理。熟悉dataview插件的朋友可以跳过这个部分。注意，这不是dataview的使用教程，只是让不熟悉dataview的朋友理解在搭建私人图书管理系统过程中，dataview的工作原理是什么样的。详细的使用教程请参考官方文档：[Dataview](https://link.zhihu.com/?target=https%3A//blacksmithgu.github.io/obsidian-dataview/)

Dataview是obsidian中的一个插件。它可以对指定文件夹中的文件（Markdown文件）进行检索，然后统计这些文件的相关信息，并以表格、列表等形式将统计结果显示出来。

那么它是通过什么来检索文件并读取文件中的信息呢？

简单说，就是通过对文件进行标记，标记的方式有两种：

- 行内标记。即在md文件中，任意行内插入[key:: value]格式的内容。
- 文件开头标记。在md文件开头插入YAML格式的字段，具体格式是在2行---之间的插入想要包含的内容。如：

对文件做了上述标记，就意味着在使用dataview的时候，此文件可以被检索，而这些字段（如书名、作者）就是这个md文件的metadata，是完成各种统计任务的基础数据。

可以插入与这个文件相关的任何信息作为字段，包含的字段越多，意味着可以分析和统计的角度越多。

## 2.1.2  为每本书建立“书名.md”文件并录入对应字段信息。

每次添加一本书就是为其建立一个单独的md文件，然后填写对应字段的信息。下面是录入信息的方法：

- 笨方法：手动录入。即全部手动输入这些信息，很直接也很好理解，缺点是信息太多，每次录入比较繁琐。
- 自动录入书籍基本信息。此方法可以从豆瓣自动导入书籍的基本信息，如书名、作者、出版社、豆瓣评分等等。剩下的字段如阅读、购买相关的信息仍需根据自己需要选择是否录入。

## 2.2.6 笔记系统

使用Obsidian记录笔记主要优势如下：

- 灵活的笔记组织方式：Obsidian采用超链接方式来组织笔记，这种方式可以更自由、更灵活地组织和管理笔记，同时也能够更直观地显示笔记之间的关系。
- 强大的搜索功能：该软件搜索功能非常强大，它可以搜索笔记标题、正文、标签等多个维度，还支持正则表达式、通配符等高级搜索方式，帮助用户快速找到所需信息。
- 多种工具的支持：Obsidian支持Markdown格式，可以轻松编辑文本，也可以使用插件来增强编辑功能。此外，还支持集成其他工具，如Git、VSCode等，使得用户可以更自由地扩展功能。
- 安全性：可以将数据保存在本地，保证了数据的安全性和隐私性。
- 跨平台支持：Obsidian支持Windows、MacOS和Linux等多个平台，使得用户可以在不同的设备上无缝使用。

有了之前为每本书创建的md文件，可以方便的在里面记录笔记，同时借助obsidian的双向链接功能很方便的建立自己的知识（笔记）系统，有利于知识、笔记的管理。如文章开头所演示的。

## 3. 总结

最后，总结一下，使用obsidian+dataview整理图书，相比于Excel表格，优势在于信息管理和数据统计更灵活；而相比于APP应用，优势在于所有数据保存在本地，不用担心丢失风险和APP停服风险。

唯一的缺点是录入信息工作比较繁琐，尤其是想要录入的字段比较多时。但字段越多，意味着分析书籍信息的角度越多，而且以后每次添加书籍，**所有表格和统计信息会自动更新**，所以长期来看，是一劳永逸的。
